Roulette

by: Ryan Boyes

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I decided to make this specific game because I thought building a casino would be a fun game that everybody would be able to play. Roulette was always a game that interested me because I think there can be some strategy to it, even though it is probably all luck. When I was first thinking of ideas, roulette was one of the first that came to me, and I decided to stick with it.

2. What I Learned

I learned a lot more making this game than I originally thought I would. Even though I had already done projects with runners and objects, I think I understand how it works far more than before. I used 7 separate classes in this game, all of which were intertwined with each other. I used objects to organize and use variables from different classes as well. My classmates helped me make some changes that they felt would make my game easier to understand and play.

3. Architecture of My Game

My game had 7 classes: Runner, GamePanel, RoulettePanel, Scoreboard, Wheel, RouletteTable, and Spin. The scoreboard, wheel, rouletteTable, and Spin classes were all added onto the roulettePanel in a box layout so they would be organized from top to bottom. This way, when the player tries to switch from the menu to the game state, all 4 of these classes will show up in an organized way on this panel. The game panel was mostly the panel for the menu state, but also allowed the game to switch states and added either the menu panel or roulette panel onto the main frame.

4. Technical Implementation

In the process of making my game, I hit a lot of roadblocks that stopped my progress. I needed a way to make the image of a roulette wheel show up in a set area, while also having room to show the entire roulette table. I solved this problem by creating multiple classes for each specific part of the table, that way I could convert the image of a wheel into a JLabel and put it on its own panel. This allowed the wheel to show up in its own location on the frame.

5. Most Challenging Part

The most challenging part of this program was trying to get the wheel and roulette table to show up in separate areas. I tried for a long time to put the image in a certain area of one big panel, but that never worked. After a long time of troubleshooting, I got the idea to convert the image into a JLabel, where I could then make it its own panel. This made it so the wheel would show up, and the separate panel would be in a different area than the RouletteTable panel. From there, I was able to organize the panels with a BoxLayout which is what gave me the design I have now.

6. Future Fixes and Updates

I tried working on adding a feature where the wheelLabel would rotate whenever the spin button was clicked, but I couldn't find a way to add this, and I felt it was unnecessary to spend more time on it because it wasn't needed for the game to work. If I had more time, I would research more ways and try to get the wheel to spin.
